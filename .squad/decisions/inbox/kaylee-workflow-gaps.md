# Workflow Gap Analysis — Strata v0.0.5 (PR #54)

**Author:** Kaylee  
**Date:** 2026-05-31  
**Scope:** haven's `.github/` vs strata's updated `deploy-workspace.yml` + new composite actions

---

## TL;DR

Haven's `deploy.yml` is a strata scaffold **template** (still has unresolved `${name}`, `${file}`, `${environment}` placeholders). It was never instantiated. The composite actions that strata's updated workflow depends on don't exist in haven at all. Six distinct gaps.

## Summary by Priority

### Must fix (broken without these)
- **Items 1, 2, 3** — the three composite actions must be created in haven's `.github/actions/`. The updated strata workflow will fail immediately without them since it uses relative paths.
- **Item 5** — `deploy.yml` has unresolved `${name}`/`${file}`/`${environment}` placeholders. Haven has no working deploy workflow right now.

### Should fix (correctness/reliability)
- **Items 4, 6, 7, 9, 13** — strata install method, artifact split removal, concurrency group, dry_run expression, missing `--work-path`.

### Needs a decision before touching (haven-specific vs strata generic)
- **Items 8, 10, 11, 12, 14** — Terraform install, `--force` flag, `--stage` input, `--deep` on validate, trigger type. These involve either confirming strata v0.0.5 behaviour or an architectural preference. Items 10, 11, 14 are haven-specific concerns that should be validated with the broader team before changing.
